# Identifier Specification

**Document:** `specifications/core/identifiers.md`

**Version:** 1.0

**Status:** Draft

---

# 1. Purpose

This specification defines identifier requirements for all persistent objects within the Archive Platform.

Identifiers provide stable references to logical objects.

Identifiers carry no application semantics.

---

# 2. Goals

Identifiers MUST provide:

- global uniqueness
- stability
- transport independence
- portability

---

# 3. Identifier Properties

Persistent identifiers MUST:

- remain stable
- never be reused
- never change

Identifiers MUST NOT encode business meaning.

---

# 4. Standard Identifiers

The platform defines identifiers for:

- Workspace
- Vault
- Bundle
- Artifact
- Event
- Device
- Session
- Protocol Message
- Block
- Wrapped Key

Future object types MAY define additional identifiers.

---

# 5. Identifier Generation

Identifiers SHALL be generated by trusted clients.

Servers MUST NOT rewrite identifiers.

---

# 6. Equality

Identifiers are compared byte-for-byte.

Comparison is case-sensitive unless the identifier format specifies otherwise.

---

# 7. Lifetime

Once assigned, an identifier remains associated with exactly one logical object.

Deleted objects do not free identifiers for reuse.

---

# 8. Scope

Identifiers are globally unique.

Object type determines interpretation.

---

# 9. Serialization

Identifiers SHALL have a canonical binary representation.

A canonical text representation MAY also be defined.

---

# 10. Unknown Formats

Readers MUST reject identifiers that do not conform to the declared identifier format.

---

# 11. Invariants

Identifiers never change.

Identifiers are never reused.

Identifiers are opaque.
