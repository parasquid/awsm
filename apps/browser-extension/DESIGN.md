# Browser Extension UI Design

The extension uses a modern-neutral archival-tool aesthetic: white and cool-gray surfaces, dark text, blue primary actions and focus rings, compact system typography, restrained borders, and minimal shadow. It uses no UI framework. Network access is optional, limited to the exact user-approved Coordination Server origin, and carries only Account authentication material and opaque encrypted synchronization data.

The popup is a 360 px task surface with explicit onboarding, locked, capturing, success, warning, and failure states. First use presents one clear primary decision and keeps custom-server configuration in progressive disclosure. Capture continues in the background if the popup closes. Reopening the popup derives its state from the persisted Runtime job. A successful recent Capture remains visible for eight seconds; a vertical progress rail fills downward to communicate when the preview will dismiss and pauses while the preview is hovered or keyboard-focused.

The library is a full extension page. Vault switching and creation are secondary management tasks inside Settings rather than primary popup or Library actions. The list contains title, source URL, capture time, screenshot availability, and warnings. Detail content is decrypted only after unlock. The full screenshot is retrieved automatically and rendered as the first content preview above the technical Artifact list through a short-lived Blob URL. Text and structured content expand directly beneath their selected Artifact row. Thumbnail has no detail action. The mandatory `PRIMARY` role is labeled **MHTML** in the interface and is the only downloadable Capture Artifact. Its verified plaintext is streamed through the Chrome Download Host using a temporary extension-owned file and Chrome's default download behavior; it is never assigned to a frame, object, embed, window location, or executable document surface.

Vault-level maintenance follows the Library content in a lightly elevated neutral panel. Reversible storage relief is described as reducing **device storage**, never as receiving “free” storage. Permanent Vault Vacuum remains visually separated and uses the destructive treatment. Remote-only Artifacts retain their supported semantic actions and show that they are stored on the server and retrieved when opened. Screenshot retrieval, text inspection, and MHTML download surface loading, retry, success, and failure states without moving unrelated Library content.

All controls are keyboard reachable, use native semantics, retain visible focus, and report asynchronous state through polite live regions. Color is never the only status signal. Motion is nonessential and reduced-motion preferences are honored.
